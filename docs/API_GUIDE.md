# VLE API Guide

This guide is for teammates who need to run, test, explain, or extend the COMP3161 VLE API. It focuses on the practical workflow rather than every implementation detail.

If you need the reasoning behind the current structure, auth choice, grade flow, or seed-data approach, read `docs/DECISIONS.md` alongside this guide.

## Project Shape

The system is a Flask API backed by MySQL. The schema lives in `vle.sql`, report views live in `reports.sql`, and deterministic demo data is generated by `gen_vle.py` into `vle_inserts.sql`.

The API is currently kept in one file, `app.py`, for simple group handoff. The file is split into helper sections and route sections:

- Validation helpers normalize input and return clean JSON errors.
- Authentication helpers use Basic Auth for protected routes.
- Lookup helpers centralize repeated database checks.
- Route groups map to the coursework requirements: users, courses, enrollments, content, forums, calendar events, assignments, grading, and reports.

## Setup

Create the virtual environment and install dependencies:

```powershell
powershell -ExecutionPolicy Bypass -File .\setup.ps1
```

Create `.env` from `.env.example`:

```env
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_NAME=Vle
```

Regenerate seed data:

```powershell
.\.venv\Scripts\python.exe gen_vle.py
```

Rebuild the database in this order:

```powershell
mysql -u root -p < vle.sql
mysql -u root -p Vle < vle_inserts.sql
mysql -u root -p Vle < reports.sql
```

Run Flask:

```powershell
.\.venv\Scripts\python.exe app.py
```

The default local API base URL is:

```text
http://127.0.0.1:5000
```

## Demo Credentials And Basic Auth

Use `demo_credentials.txt` for local Postman/demo accounts. The generated seed data deliberately includes:

- Student `620000000`
- Lecturer `200000000`
- Admin `111000000`

Protected routes use Basic Auth. In Postman, select the Authorization tab, choose Basic Auth, and enter the account email and password from `demo_credentials.txt`.

The public `POST /auth/login` route is still useful as a credential check, but protected routes do not use a token. They verify Basic Auth directly on each request.

## Route Groups

Public/demo routes:

- `GET /api/data`
- `GET /courses`
- `GET /users`
- `GET /users/{userID}`
- `POST /auth/login`
- Report routes under `/reports/...`

Admin-only management routes:

- `POST /users/register`
- `POST /courses`
- `PUT /courses/{courseCode}/lecturer`
- `POST /courses/{courseCode}/enrollments`

Course management routes for admins or assigned lecturers:

- `POST /courses/{courseCode}/sections`
- `POST /sections/{secID}/items`
- `POST /courses/{courseCode}/forums`
- `POST /courses/{courseCode}/calendar-events`
- `PUT /submissions/{subID}/grade`

Course read routes for admins, assigned lecturers, or enrolled students:

- `GET /courses/{courseCode}/members`
- `GET /courses/{courseCode}/sections`
- `GET /courses/{courseCode}/content`
- `GET /courses/{courseCode}/forums`
- `GET /forums/{dfID}/threads`
- `GET /threads/{dtID}`
- `GET /courses/{courseCode}/calendar-events`

Student-owned routes:

- `GET /students/{userID}/courses`
- `GET /students/{userID}/grades`
- `GET /students/{userID}/{courseCode}/grades`
- `GET /students/{userID}/calendar-events`
- `POST /assignments/{secItemID}/submissions`

Lecturer-owned routes:

- `GET /lecturers/{userID}/courses`

## Typical Demo Workflow

1. Log in with the seed admin using `POST /auth/login`.
2. Use admin Basic Auth to register a new student.
3. Use admin Basic Auth to create a course and assign a lecturer.
4. Use admin Basic Auth to enroll the student in the course.
5. Use lecturer or admin Basic Auth to create sections and course content.
6. Use lecturer or admin Basic Auth to create a forum and calendar event.
7. Use student Basic Auth to read course content and submit an assignment.
8. Use lecturer or admin Basic Auth to grade the submission.
9. Check the student's grades and the report endpoints.

## Postman Workflow

Import both Postman files:

- `postman/VLE API Full Workflow.postman_collection.json`
- `postman/VLE Local.postman_environment.json`

Select the `VLE Local` environment before running requests. The collection creates test data, so reload the database before a clean demo run if you want predictable results.

## Seed Data Rules

The generator creates data to satisfy the coursework constraints:

- 100,000 students
- 200 lecturers
- 10 admins
- 300 courses
- Each student takes 3 to 6 courses
- Each course has at least 10 student members
- Each course has exactly one lecturer
- Each lecturer teaches 1 to 5 courses
- Course grades in `Enrol.grade` are derived from graded assignment submissions

`CourseMember` is a view derived from `Enrol` and `Teaches`, so the insert file should not contain `INSERT INTO CourseMember`.

## Known Limitations

- Basic Auth is temporary and intentionally simpler than JWT for Postman testing.
- Password reset and password change are not implemented.
- Uploaded files are represented as text metadata, not real binary file storage.
- Admin-created accounts return the generated plaintext password once; it cannot be recovered later.
- The app is still a single-file Flask API for handoff simplicity. A later production-style cleanup could split routes into Blueprints.
